# 대사/나레이션 시스템

## 현재 상태

현재 빌드에서는 캐릭터 대사를 사용하지 않는다.

- 화면에 보이는 문장은 모두 시스템 나레이션이다.
- 나레이션은 `누가 무엇을 했는지`와 `지금 어떤 단계인지`만 짧게 설명한다.
- LLM은 행동 선택, 카드 교체, 라운드 회고만 담당한다.

## 구현 위치

- [poker_dialogue.rpy](../llmoker/game/poker_dialogue.rpy)
  - 행동 요약과 단계 나레이션을 만든다.
- [poker_minigame.rpy](../llmoker/game/poker_minigame.rpy)
  - 각 단계가 끝날 때 나레이션 이벤트를 호출한다.

## 나레이션 이벤트

- `match_intro`
  - 매치 시작 안내
- `round_start`
  - 라운드 시작과 선공 안내
- `betting`
  - 방금 적용된 체크, 베팅, 콜, 레이즈, 폴드 요약
- `draw`
  - 드로우 종료와 두 번째 베팅 시작 안내
- `round_end`
  - 라운드 승자와 팟 요약
- `match_end`
  - 최종 승자 안내

## 문장 규칙

- 주체 표기는 `플레이어`, `사야`로 통일한다.
- 행동 표기는 아래처럼 고정한다.
  - `플레이어는 체크했다.`
  - `사야는 10칩 베팅했다.`
  - `플레이어는 10칩 콜했다.`
  - `사야는 총 20칩까지 레이즈했다.`
  - `플레이어는 폴드했다.`
  - `플레이어는 카드 2장을 교체했다.`
- 라운드 시작과 드로우 전환도 설명문으로만 처리한다.

## 의도

이 레이어의 목적은 심리전 RP가 아니라 규칙 흐름을 읽기 쉽게 만드는 것이다.

- 지금 누가 먼저 행동하는지
- 방금 누가 어떤 행동을 했는지
- 이제 어느 단계로 넘어갔는지

이 세 가지만 확실히 보이게 유지한다.
