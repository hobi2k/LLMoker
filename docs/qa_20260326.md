## 가장 먼저 고칠 문제

### 1. 정책 회고가 여전히 전략을 망치고 있다

상태: `미해결`

관찰:

- 회고가 한 번 저장된 뒤 다음 라운드 행동에 과잉 일반화되어 들어간다.
- 로그 예시:
  - `이번 판은 상대 베팅을 받은 상황 쇼다운에서 하이카드로 졌다. 비슷한 흐름에서는 콜을 더 신중히 고른다.`
  - 이후 원페어, 투페어 상황에도 비슷한 문장을 반복 반영
- 강한 패에서도 `콜을 신중히`, `폴드를 먼저 검토` 같은 전략이 그대로 들어가 행동이 무너진다.
- 이번 로그에서는 투페어인데 폴드하는 비정상 행동까지 재현됐다.

영향:

- 라운드 2 이후 사야 행동이 실제 손패 강도보다 이전 회고 문장에 과도하게 끌려간다.
- 플레이 경험상 “AI가 생각한다”가 아니라 “한 번 잘못 배운 말을 계속 반복한다”처럼 보인다.

해야 할 일:

- 정책 회고를 현재 라운드 사실과 손패 강도 기준으로 더 세분화할 것
- `하이카드`, `원페어`, `투페어 이상`에서 허용 가능한 전략 변화 폭을 다르게 제한할 것


### 2. 카드 교체 판단이 족보를 부수고 있다

상태: `미해결`

관찰:

- 사야가 트리플이나 원페어를 들고 있어도 구성 카드를 버리는 경우가 반복된다.
- 로그 예시:
  - `다이아 Jack, 스페이드 Jack, 클로버 Jack` 트리플 상태에서 `버릴 인덱스: [0, 2]`
  - 결과: 드로우 후 `하이카드`로 추락
- 추가 로그에서도 `원페어` 구성 카드 자체를 버리는 선택이 반복적으로 나온다.

영향:

- 드로우가 전략이 아니라 실수처럼 보인다.
- 플레이어 입장에서는 사야가 기본 포커 규칙도 못 지키는 것처럼 보인다.

해야 할 일:

- 드로우 단계에서 현재 족보 구성 카드를 절대 버리지 않는 하드 제약을 넣을 것
- `원페어`, `투페어`, `트리플 이상`은 규칙 기반 보호 로직을 먼저 적용하고, 그 안에서만 LLM이 판단하게 할 것
- 교체 이유 문장도 실제로 유지한 카드와 버린 카드가 일치하는지 검증할 것


### 3. 행동 이유가 현재 상황과 맞지 않거나 자기모순적이다

상태: `미해결`

관찰:

- 이유 문장이 현재 테이블 상태를 제대로 반영하지 못한다.
- 로그 예시:
  - `상대가 10칩을 콜했으며`라고 쓰지만 실제로는 상대가 `bet`한 상황
  - `상대의 하이카드 흐름과 유사한 상황`처럼 현재 패와 무관한 추상 문장 반복
  - `최근 전략 피드백에 따라 ...`를 붙이지만 실제 행동 변화 근거가 빈약함
- `check`밖에 허용되지 않는 상황에서도 “폴드를 먼저 검토” 같은 설명이 이유에 섞인다.

영향:

- 플레이어가 로그를 봐도 왜 그 행동을 했는지 이해할 수 없다.
- AI 설명 가능성이 아니라 헛소리로 느껴진다.

해야 할 일:

- 모델에 넣는 프롬프트에 당신과 사야라는 주어 지칭에서 혼란이 없게 만들 것
  - 모델은 당신이 자신을 지칭하는 사용자를 지칭하는지 알지 못하며, 사야라는 단어 또한 마찬가지다
  - 모델에게 혼란을 주지 않도록 당신(사야)와 사용자(플레이어)로 역할 헌법을 정확히 한다
- `상대가 bet했는지`, `call했는지`, `현재 to_call이 얼마인지`를 이유 검증 단계에서 체크할 것


## 게임 흐름

### 4. 마지막 행동 후 결과 화면으로 가기 전에 나레이션이 항상 충분히 보이지 않는다

상태: `부분 해결`

관찰:

- 최근 수정으로 마지막 행동 뒤 나레이션이 먼저 나오도록 바뀌었지만, 이번 영상 기준으로는 여전히 결과 전환이 급하게 느껴지는 구간이 있다.
- 특히 마지막 `check`, `call`, `fold` 직후 결과 패널로 넘어갈 때 “무슨 행동으로 종료됐는지”를 충분히 인지하기 어렵다.

해야 할 일:

- 마지막 행동 나레이션과 결과 화면 사이에 짧은 템포를 둘 것
- `두 사람 모두 체크했다`, `사야가 폴드했다` 같은 종료 사유 문장을 더 명시적으로 넣을 것
- 폴드 종료와 쇼다운 종료를 화면 연출에서도 다르게 느껴지게 만들 것


## UI / 연출


### 8. 도움말 화면은 여전히 조밀하고 읽기 피곤하다

상태: `미해결`

관찰:

- 영상 기준으로 도움말 패널 텍스트 밀도가 높고, 스크롤 없이 한 화면에 많은 내용을 몰아넣은 느낌이다.
- 규칙을 찾기보다 “설명문 덩어리”로 먼저 보인다.

해야 할 일:

- `기본 조작`, `게임 진행 규칙`, `베팅 규칙`, `드로우 규칙`, `결과 읽는 법`으로 더 명확히 나눌 것
- 한 블록 당 줄 수를 줄이고, 핵심 규칙을 먼저 보이게 만들 것
- 실제 플레이 예시 한 판을 짧게 넣어 이해를 돕는 방식도 검토할 것


### 9. 결과 화면은 읽을 수는 있지만 여전히 정보가 빽빽하다

상태: `미해결`

관찰:

- 영상 후반 결과 패널은 이전보다 구조는 잡혔지만, 카드 줄과 요약 문장이 여전히 밀집돼 있다.
- 승패 요약, 카드, 버튼이 한 번에 몰려 보여 결과를 빠르게 읽기 어렵다.

해야 할 일:

- 종료 사유, 승자, 팟, 손패 영역의 우선순위를 더 분명히 줄 것
- 카드 줄 간격과 하단 버튼 간격을 더 벌릴 것
- 폴드 종료와 쇼다운 종료 레이아웃을 분리할지 검토할 것


## LLM / 전략 / 로그

### 10. 회고 문장 렌더링이 아직 어색하다

상태: `부분 해결`

관찰:

- 조사 중복 문제는 일부 수정됐지만, 문장 구조는 여전히 기계적이다.
- 예시:
  - `상대 베팅을 받은 상황에서는 하이카드면 비슷한 흐름에서는 콜을 더 신중히 고른다.`
  - `상대 베팅을 받은 상황에서는 원페어면 비슷한 흐름에서는 무리한 콜보다 폴드를 먼저 검토한다.`
- 의미는 읽히지만 자연스러운 한국어 전략 문장으로는 어색하다.

해야 할 일:

- `상황 + 패 강도 + 다음 원칙`을 더 자연스럽게 조립하는 별도 렌더러를 만들 것
- `비슷한 흐름에서는` 같은 연결구를 반복하지 않도록 템플릿을 다양화할 것


### 11. 로그상 행동과 전략 사이의 연결이 여전히 약하다

상태: `미해결`

관찰:

- `최근 전략: N / 장기 기억: M`은 보이지만, 어떤 전략이 실제로 선택에 영향을 줬는지는 여전히 불명확하다.
- 현재 이유 문장은 전략을 인용하는 척하지만, 실제로는 현재 손패/압박보다 전략 문장 반복에 가까운 경우가 많다.

해야 할 일:

- 행동 판단 로그에 실제 사용한 최근 전략 1개와 장기 기억 1개를 같이 찍을 것
- “선택 이유”와 “참고 전략”을 로그상 분리할 것


### 12. 반복 전략이 강한 패까지 오염시키고 있다

상태: `미해결`

관찰:

- 이번 로그 후반에서 원페어, 투페어에서도 `상대 베팅을 받은 상황에서는 ... 폴드/콜 신중`이 반복된다.
- 심지어 투페어에서 폴드까지 나와 전략 오염이 명백하다.

해야 할 일:

- 회고 저장 전 `패 강도 대비 허용 전략` 검증을 더 강하게 할 것
- 투페어 이상에서 `fold_carefully` 같은 조합은 차단할 것
- 원페어와 하이카드의 전략 분리도 더 강하게 둘 것


### 13. 카드 교체 이유와 실제 교체 카드가 계속 어긋난다

상태: `미해결`

관찰:

- 로그에서 이유 문장은 `다이아 7과 다이아 4를 교체`라고 하지만 실제 버린 인덱스와 손패 구성이 안 맞는 구간이 있다.
- 강한 패 유지라는 말과 실제 행동이 충돌하는 경우가 계속 나온다.

해야 할 일:

- 드로우 이유 문장은 선택된 인덱스를 실제 카드 이름으로 다시 역매핑한 뒤 조립할 것
- 이유 생성 전에 “선택 인덱스가 현재 족보 구성 카드를 포함하는지” 검증할 것
