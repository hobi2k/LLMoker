## 확인한 문제

### 1. 시작 연출

상태: `부분 해결`

- `openingcinema -> main menu` 전환이 어색했다.
- 컷신 뒤 메뉴가 갑자기 올라오는 느낌이 있었다.
- `logo` 뒤 안내 문구가 빠져 있었다.

조치:
- `splashscreen`에 안내 문구를 넣었다.
- `openingcinema.webm` 뒤 메인 메뉴 배경으로 페이드하는 흐름을 넣었다.
- 메뉴 패널과 타이틀의 등장 타이밍을 분리했다.

상태:
- 개선됨
- 다만 `openingcinema.webm` 원본 프레임 특성 때문에 완전히 영화적인 연결감은 아직 아니다.

### 2. LLM 예열

상태: `부분 해결`

- 기존에는 `label start`에서 동기 예열 대기가 남아 있었다.
- 사용자 체감상 “시작 직전 멈춤”처럼 느껴질 여지가 있었다.

조치:
- `begin_llm_npc_prewarm()`를 `splashscreen` 시작 시점에 호출하도록 유지했다.
- `finish_llm_npc_prewarm()`를 추가해 스플래시 안에서 남은 예열을 마무리하게 바꿨다.

상태:
- 개선됨
- 이제 예열 대기 구간은 `start`보다 스플래시 안에 더 많이 들어간다.

### 3. 정책 회고 실패

상태: `부분 해결`

관찰 로그:
- `정책 피드백 생성 실패: 'NoneType' object has no attribute 'get'`

조치:
- `policy_loop.py`에서 회고 응답 형식과 예외를 한 번 더 감쌌다.
- `runtime.py`에서 회고 JSON 필드 읽기를 `None` 안전하게 바꿨다.
- `runtime.py`에서 비정상 메시지 객체를 만났을 때 `role`과 `function_call` 속성 접근을 `getattr` 기반으로 다시 감쌌다.
- `agent.py`에서 회고 필드가 `None`이어도 `"None"` 문자열로 저장되지 않게 `or ""` 정리를 넣었다.

상태:
- 플레이어 경험 기준으로는 덜 치명적으로 바뀌었다.
- 다만 왜 `None`이 들어오는지의 근본 원인 추적 로그는 더 필요하다.

### 4. 대사 품질

상태: `부분 해결`

관찰:
- 대사가 설명투, 프롬프트 냄새, 반복 표현을 보였다.
- 상태 설명을 어색한 한국어로 흉내 내는 문장이 많았다.

조치:
- 대사 프롬프트를 더 직접적이고 짧게 정리했다.
- 이벤트 설명을 더 짧고 구체적인 사건 기준으로 바꿨다.
- 내부 LLM 판단 로그가 사용자 공개 흐름에 섞이지 않게 정리했다.

상태:
- 개선 중
- 여전히 추가 튜닝이 필요하다.

### 5. 행동/교체 이유

상태: `부분 해결`

관찰:
- 이유 문장이 지나치게 일반론적이었다.
- 카드 교체가 `[0, 2]` 패턴으로 반복되는 경향이 강했다.

조치:
- 행동 이유는 팟, 콜 금액, 스택, 족보 중 하나를 직접 언급하게 프롬프트를 조정했다.
- 카드 교체 이유는 카드 내용이나 족보 부족 지점을 직접 언급하게 조정했다.

상태:
- 개선 중
- 패턴 반복 여부는 추가 플레이 로그로 재검증이 필요하다.
